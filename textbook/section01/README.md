# セクション1：AIで作ったコードを理解できるようになるために

## このセクションの目的

「AIで作れるのに、なぜバックエンドを学ぶ必要があるのか？」

この問いに正面から答えるのが、このセクションの目的です。  
知識を詰め込む前に、**学ぶ理由を腹落ちさせる**ことを最優先にします。

---

## AIで開発して感じる限界

### 成功体験から始まる

AIを使えば、驚くほど簡単にアプリが作れます。

「ログイン機能を作って」  
「データを保存できるフォームを作って」

こう伝えるだけで、それらしいコードが出てきます。  
最初は感動するはずです。

### でも、こんな場面に直面します

しばらくすると、壁にぶつかります。

**よくある問題：**

- **エラーが直せない**  
  エラーメッセージが出ても、どこを直せばいいか分からない。  
  AIに聞くと別の問題が出てきて、泥沼になる。

- **ちょっと変えたいけど無理**  
  「この部分だけ変えたい」と思っても、どこを変えればいいか分からない。

- **コードの意味が分からない**  
  動いてはいるけど、何をしているコードなのかが読めない。

- **どこを直せばいいか分からない**  
  問題は起きているのに、原因の場所すら特定できない。

### 本質の問い

なぜ、こういうことが起きるのでしょうか？

> **AIは「実装」はできますが、「理解」は補えません。**

AIはコードを生成できます。  
でも、あなたがそのコードを理解しているかどうかは、AIには関係ありません。  
理解がないまま使い続けると、問題が起きたときに何もできなくなります。

### 問題の正体

つまり、問題の本質はこれです。

1. **構造を理解していない** — どこに何があるか分かっていない
2. **バックエンドを知らない** — サーバー側で何が起きているか見えていない
3. **データの流れが分からない** — リクエストがどこを通って、どう返ってくるか追えない

### この講座で扱う問題

この講座で扱うのは、まさに「動くけど直せないコード」です。
AIに頼ること自体が悪いのではありません。
問題は、コードがブラックボックス化してしまうことです。

> **メッセージ：**  
> AIを使いこなすには、基礎が必要です。  
> 基礎があってこそ、AIは最強のツールになります。

この講座では、名前・メールアドレス・電話番号を保存できる簡易的なアドレス帳アプリを作りながら、
画面、API、データベースがどうつながるかを順番に理解していきます。

---

## フロントエンドだけでは不十分な理由

### フロントエンドでできること

フロントエンド（HTML・CSS・JavaScript）だけでも、以下は作れます。

- 画面を表示する
- ボタンを押したときの動作を作る
- 入力フォームを作る

### フロントエンドだけでできないこと

フロントエンドだけでも、ブラウザ内に一時的・ローカルな保存はできます。
しかし、次のようなことは苦手です。

- 別の端末から同じデータを見る
- 複数ユーザーで同じデータを共有する
- 安全にユーザー情報を管理する
- データをサーバー側で一元管理する

### 結論

本当の意味でアプリとして機能させるには、**バックエンドが必要**です。

---

## バックエンドとは何か

### シンプルに理解する

```text
フロントエンド ➡️ バックエンド ↔️ DB
       ↖️ 結果を返す ↙️
```

それぞれの役割はこうです。

| 層 | 役割 | アドレス帳の例 |
|---|------|--------------|
| フロントエンド | 見た目・操作 | 入力フォーム・一覧画面 |
| バックエンド | 処理・制御 | 保存する・取得する・削除する |
| データベース（DB） | データの保存 | 連絡先データそのもの |

### 具体例：アドレス帳で考える

「新しい連絡先を登録する」という操作を例にします。

1. **フロントエンド** — フォームに名前・メールアドレスを入力して送信ボタンを押す
2. **バックエンド** — 受け取ったデータをチェックし、DBへ保存する命令を出す
3. **データベース** — データを永続的に保存する

この3層の役割を理解することが、バックエンド開発の出発点です。

### なぜこの図が重要なのか

この図が頭に入ると、以下のことが見えてきます。

- エラーが起きたとき、**どの層の問題か**を特定できる
- AIが生成したコードが、**どの部分を担当しているか**が分かる
- 「何を修正すればいいか」が判断できるようになる

> **ここで一気に視界が開けます。**

## コースの概要

### この講座で作るもの

この講座では、**簡易アドレス帳アプリ**を一から作ります。

アドレス帳というとシンプルに見えますが、扱う内容は実務アプリの基本そのものです。  
名前、メールアドレス、電話番号などの情報を登録・検索・更新・削除する流れは、顧客情報管理、会員管理、アカウント管理、予約管理など、多くのWebアプリに共通しています。

つまり、この講座で作るアドレス帳は、単なる練習用アプリではありません。  
**「人や組織に関するデータを管理するアプリ」の最小版**として考えることができます。

- 連絡先を登録・表示・編集・削除できる
- シンプルだからこそ、本質だけに集中できる

### 使う技術

| 技術 | 役割 |
|------|------|
| PHP | バックエンド（サーバー側の処理） |
| SQLite | データベース（データの保存） |
| VSCode | 開発環境（コードを書くエディタ） |

### 対象者

- AIでアプリを作ったことはある
- でも、コードの意味はよく分からない
- エラーが出ると、どこを直せばいいか分からない

そんな方に向けた講座です。  
プログラミング経験が浅くても大丈夫です。
HTMLやJavaScriptを少し触ったことがある、またはAIでコードを生成したことがある人なら、順番に理解できる構成にしています。

### AI活用について

この講座では、AIを使ってコードを生成する場面も扱います。  
ただし、ゴールは「AIに全部やってもらうこと」ではありません。  
**AIが生成したコードを理解し、自分で修正できる状態**を目指します。

---

## この講座でできるようになること

この講座を受講すると、以下ができるようになります。

### 1. APIが作れる

フロントエンドからデータを受け取り、処理して返す「窓口」を自分で実装できます。

### 2. データベースが扱える

データの保存・取得・更新・削除を、SQLを使って操作できます。

### 3. フロントエンドと繋げられる

JavaScriptから自作のAPIを呼び出し、アプリとして動かすことができます。

### 4. 「なぜ動くか」が分かる

コードを書くだけでなく、**なぜそう書くのか**を説明できる状態になります。

> **ポイント：「作れる」より「理解できる」**
>
> AIがあれば、コードは作れます。  
> でも、「なぜそう動くのか」は、自分で理解するしかありません。  
> この講座がフォーカスするのは、その「理解」の部分です。
