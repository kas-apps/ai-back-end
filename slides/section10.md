# AIにコードを生成させてみる

---

## 実際にやってみる

セクション6で作ったのと同じAPIを、AIに生成させてみます。

```text
PHPとSQLiteを使って、連絡先を管理するAPIを作ってください。
GET/POST/PUT/DELETEに対応してください。
DB接続は api/db.php に分け、API本体は api/index.php に書いてください。
```

AIは動くコードを生成します。  
しかし、出力されたコードをそのまま使う前に、確認すべきことがあります。

---

## 生成されたコードを確認する

セクション9で学んだチェックポイントを使って確認します。

- **プリペアドステートメントを使っているか**  
  文字列連結でSQLを組み立てていないか
- **`innerHTML` を使っていないか**  
  フロントエンドも生成させた場合、表示処理に注意
- **バリデーションが入っているか**  
  `name` が空のまま登録できないか
- **エラー情報をそのまま返していないか**  
  `catch ($e) { echo $e->getMessage(); }` は内部情報が漏れる
- **認証・認可が必要な操作を誰でも実行できないか**  
  追加・更新・削除APIが無条件で呼び出せる状態になっていないか

<!-- AIが生成したコードは「動く」かもしれませんが、「安全」とは限りません。判断するのは、コードを読む人間です。 -->

# プロンプトで結果は変わる

---

## 曖昧なプロンプトと具体的なプロンプト

同じ目的でも、プロンプトの書き方によってAIの出力は大きく変わります。

#### **曖昧なプロンプト**

```text
PHPでAPIを作って
```

セキュリティ対策・バリデーション・エラーハンドリングが欠けたコードが生成される可能性があります。

#### **具体的なプロンプト**

```text
以下の条件でPHPのAPIを作ってください。
- SQLiteを使う
- PDOとプリペアドステートメントを使う
- nameが空の場合は400エラーを返す
- エラーメッセージに内部情報を含めない
- api/index.php と api/db.php に分ける
```

<!-- **AIに「何を作るか」だけでなく「どう作るか」を伝えることが重要です。** -->

---

## プロンプトに含めると良いこと

| 項目 | 例 |
|------|---|
| 使用技術 | PHP / PDO / SQLite |
| セキュリティ要件 | プリペアドステートメントを使う |
| バリデーション要件 | name の入力は必須、空なら400エラー |
| エラー処理の方針 | 内部情報をレスポンスに含めない |
| ファイル構成 | `api/index.php` と `api/db.php` に分ける |
| 認証・認可 | 追加・更新・削除には本来認証が必要であることを明記する |

これらを事前に伝えておくと、後から修正する手間が減ります。

---

## より良いプロンプト例

セクション8・9の内容まで反映させたプロンプトです。

```text
PHPとSQLiteを使って、連絡先を管理するAPIを作ってください。

条件：
- GET /api/contacts で一覧取得
- POST /api/contacts で登録、PUT /api/contacts/{id} で更新
- DELETE /api/contacts/{id} で削除
- DB接続は api/db.php の get_pdo() に切り出す
- SQLはPDOのプリペアドステートメントを使う
- nameが空の場合は400エラーを返す
- レスポンスに内部エラー情報を含めない
- 追加・更新・削除APIには本来認証・認可が必要であることをコメントで明記する
```

フロントエンドも生成させる場合は、次の条件も加えます。

```text
- ユーザー入力を表示するときに innerHTML を使わず、textContent を使う
```

# AIの限界を知る

---

## AIが得意なこと

AIは次のようなことを得意としています。

- 決まったパターンのコードを生成する
- ドキュメントや既存のコードをもとに説明する
- 繰り返し作業を素早くこなす
- 複数の選択肢を出す

セクション3から6で作ったPHPのAPIも、AIに生成させると似たようなコードが得られます。

---

## AIが苦手なこと

| 苦手なこと | 内容 |
|-----------|------|
| 文脈の判断 | 「誰でも削除できてよいか」はアプリの要件次第。<!-- AIはその要件を知らない --> |
| 設計の判断 | 責務分離のような方針は明示しないと反映されない |
| セキュリティの保証 | セキュリティ対策のコードを生成できるが、必ず安全とは言えない |
| 動作確認の責任 | 実際に自分の環境で動くかどうかまでは保証しない |

<div class="important">

> **AIはコードを書けますが、責任は取りません**。動かす判断をするのは、いつも人間です。

</div>

# AIにレビューさせる

---

## AIにレビューを依頼する

AIはコード生成だけでなく、既存コードのレビューにも使えます。

#### `api/index.php` を確認させるなら

```text
以下の api/index.php をレビューしてください。
特に、SQLインジェクション、XSS、認証・認可不足、
バリデーション不足、エラー処理の観点で問題を指摘してください。
```

#### `app.js` を確認させるなら

```text
以下の app.js をレビューしてください。
特に、innerHTMLによるXSSリスク、fetchのエラー処理、
フォーム入力値の扱いに問題がないか確認してください。
```

<div class="important">

> ただし、AIの指摘もそのまま正しいとは限りません。セクション8・9で学んだ観点と照らし合わせて、指摘が妥当かどうかを自分で確認します。

</div>

# 人間とAIの役割分担

---

## 何を任せてよいか

| 作業 | 担当 | 理由 |
|------|------|------|
| CRUDのコード生成 | AI向き | パターンが決まっている |
| SQLの雛形作成 | AI向き | 構文の知識があれば生成しやすい |
| コメント・ドキュメント生成 | AI向き | 既存コードをもとに説明できる |
| セキュリティの確認 | 人間 | コードの意図と文脈を理解する必要がある |
| 動作確認 | 人間 | 自分の環境で本当に動くか確認する必要がある |
| 設計の判断（責務・分離） | 人間 | アプリの要件・背景を知っている |
| バリデーション要件の定義 | 人間 | 何が「正しいデータか」はビジネスルール次第 |
| 認証・認可の設計 | 人間 | 誰に何を許可するかはアプリの要件次第 |

---

## AIを使うサイクル

```text
1. 要件を自分で整理する
   ↓
2. AIにコードを生成させる（具体的なプロンプトで）
   ↓
3. 生成されたコードを読んで理解する
   ↓
4. セキュリティ・設計を確認する
   ↓
5. 動作確認する
   ↓
6. 問題があれば修正する（AIに依頼するか、自分で直す）
```

「生成→確認→修正」のサイクルを回すことが重要です。

<div class="tip">

> **AIは「下書き」を作るのが得意です**。下書きを最終的な成果物にするのは、読んで判断できる人間の仕事です。

</div>

# コードを読む力がなぜ必要か

---

## 「動く」と「分かる」は違う

このコースの冒頭で、次の問題を紹介しました。

- 「AIが作ったコードは動く。でも、エラーが出たときに何が起きているか分からない。」

ここまでのセクションで、次のことを学んできました。

| セクション | 学んだこと |
|-----------|-----------|
| 2・3 | HTTPリクエストがどう処理されるか |
| 4・5 | データがどこに保存されるか |
| 6 | PHPがどうデータベースに接続するか |
| 7 | フロントエンドとバックエンドがどうつながるか |
| 8 | コードをどう整理するか |
| 9 | どんな攻撃があり、なぜ対策が必要か |

---

<!-- _class: tight -->

## 知識があるとAIをより活用できる

知識が無いままAIを使うと、次のような問題が起きやすくなります。

- 生成されたコードが正しいかどうか判断できない
- エラーが出たとき、何を直せばよいか分からない
- AIに間違ったことを聞いてしまう（前提が間違っていると答えも間違う）

逆に、基礎知識があると次のことができます。

- 「このコードはプリペアドステートメントを使っているか」を確認できる
- エラーメッセージから原因を読み取れる
- AIに「この書き方の問題点は何か」と聞いて答えを評価できる

<div class="important">

> **AIを使いこなすには、AIに頼らない判断力が必要です**。セクション1から9で学んだ内容は、その判断力の土台です。

</div>
